# LaTeX → HTML5 via TeX4ht (`make4ht`)

Experiments converting LaTeX sources to accessible HTML5 using
[TeX4ht](https://www.tug.org/tex4ht/) (driven by `make4ht`), with a
small post-processing pass that adds `role="presentation"` to the
equation-layout tables tex4ht emits.

This directory mirrors the structure of the sibling `latexml/` pipeline:
hand-written `config.cfg` and `style.css` at the root, `build.sh` and
`fix-a11y.sh` orchestrating the build, and example sources/outputs
under `examples/`.

## Examples

| Folder | What it shows |
|---|---|
| [`examples/minimal/`](examples/minimal/) | Smallest possible smoke test — `article` class, one display equation, prose filler. Verifies the `make4ht` toolchain. |
| [`examples/accessibility/`](examples/accessibility/) | The same `test_accessibility.tex` source used in `latexml/examples/accessibility/` and `tagging/examples/full-document/`: theorems, Euler/Gaussian/residue/Stokes, a TikZ figure with `alt=` text. Each example folder commits **both** a MathJax-rendered and a MathML-rendered HTML so the two math-mode renderings can be compared directly. |

## How to compile

From any example folder:

```sh
cd examples/<example-folder>/
../../build.sh [-m mathjax|mathml] <basename>.tex
```

`build.sh` wraps `make4ht -c $SCRIPT_DIR/config.cfg -f html5 -d $output_dir -s <input>.tex <math_mode>`, then runs `fix-a11y.sh` on the resulting HTML.

The default math mode is **MathML** (native browser rendering, no
JavaScript). The output filename encodes the math mode automatically:

```sh
../../build.sh                 test_accessibility.tex   # → test_accessibility.mathml.html
../../build.sh -m mathjax      test_accessibility.tex   # → test_accessibility.mathjax.html
```

so building both variants is just two invocations.

## Two math-rendering modes

| Mode | How it works | Output characteristics |
|---|---|---|
| **MathML** (default) | tex4ht emits native MathML inside the HTML | No JavaScript needed; renders directly in modern browsers (Firefox always; Chrome ≥ 109; Safari). Smaller HTML. Equations land in `<table class='equation'>` layout containers, which `fix-a11y.sh` marks with `role="presentation"` so screen readers don't announce them as data tables. |
| **MathJax** | tex4ht emits LaTeX-source-with-MathJax-CDN-loader markup | Wider browser compatibility (legacy browsers); pretty rendering; requires JavaScript to load the MathJax library at view time. Does not emit equation tables, so the `role="presentation"` fix has nothing to do. |

The committed `examples/accessibility/` folder ships both variants so
they can be compared side-by-side.

## Custom styling

`config.cfg` and `style.css` at the root of this directory are the
canonical hand-written files and are tracked in git:

- **`config.cfg`** — tex4ht configuration that wraps `theorem`,
  `definition`, and `corollary` environments with HTML5 `<div
  class="newtheorem ...">` markers (so the styling can target them
  semantically), plus a `\Configure{AddCss}{style.css}` directive that
  links our stylesheet from the generated HTML.
- **`style.css`** — responsive theme CSS with light/dark-mode custom
  properties, theorem-box styling, math-rendering tweaks, and
  print/mobile media queries.

Per-example copies of `style.css` and the per-document CSS that tex4ht
emits (e.g. `test_accessibility.css`) are *byproducts* of the build —
`build.sh` copies the canonical `style.css` into each example folder
before running `make4ht` so it can be found at compile time. These
copies are regenerated by every build and are gitignored. To adjust
styling, edit the canonical files at the `tex4ht/` root and re-run
`build.sh`.

## Prerequisites

- **TeX4ht / `make4ht`** — `brew install make4ht` on macOS, or your
  TeX-distribution equivalent.
- **TeX Live** — `make4ht` shells out to LaTeX for the underlying
  compilation.
- A modern browser with native MathML rendering (Firefox always;
  Chrome ≥ 109; Safari) for inspecting the MathML variant. The
  MathJax variant works in any browser with JavaScript enabled.
